# 03 · 周期三：用户体系与"数字烟花工坊"

## 3.1 渐进式鉴权（Progressive Auth）策略

### 无感体验设计原则

**核心原则**游客状态下开放一切"看"的能力，只在触发"存"或"得"的核心价值行为时，由花傩以拟人化语气引导登录。绝不在进站时强制登录。

 

### 鉴权触发矩阵

| 用户行为               | 是否需要登录 | 触发方式                                         |
| ---------------------- | ------------ | ------------------------------------------------ |
| 浏览地图、查看景点     | 否           | 直接可用                                         |
| 向花傩提问（基础问答） | 否           | 直接可用                                         |
| 体验数字烟花画布       | 否           | 直接可用                                         |
| 保存烟花配方           | 是           | 花傩弹出登录框：「我帮你把这场烟花永久留住～」   |
| 领取优惠券             | 是           | 花傩弹出登录框：「先跟我登记一下，券才好归你～」 |
| 打卡非遗记录           | 是           | 花傩弹出登录框：「傩愿你留下足迹！」             |
| 分享个人烟花           | 是           | 系统提示需登录后方可生成分享链接                 |

 

### JWT 认证流程规范

10. 用户登录 → NestJS 签发 Access Token（15分钟）+ Refresh Token（7天，存 HttpOnly Cookie）
11. 前端 Axios 请求拦截器自动附带 Authorization: Bearer {token}
12. Access Token 过期 → 自动用 Refresh Token 换新 Token（透明续签）
13. 用户登出 → 将 JWT jti 写入 Redis 黑名单，Refresh Token Cookie 清除
