## 一、P0 — 无障碍快速修复

> 当前无障碍平均分 95.7，以下列出所有 Lighthouse 检测到的具体问题，修复后目标 98+。

### 全局问题（所有页面）

#### 1.1 语言切换按钮标签不匹配

**位置**：[Navbar.vue](packages/frontend/src/components/Navbar.vue) 语言切换按钮

**问题**：按钮可见文字为 "EN"，但 `aria-label` 为"切换到英文"。屏幕阅读器读出的内容与可见文字不一致，违反 WCAG 2.5.3。

**修复**：使 `aria-label` 与可见文本一致：

```html
<!-- 修复前 -->
<button aria-label="切换到英文">EN</button>

<!-- 修复后 -->
<button :aria-label="$t('navbar.switchLang')" lang="en">EN</button>
<!-- 其中 navbar.switchLang: "Switch to English" / "切换到中文" -->
```

#### 1.2 "了解更多" 重复链接文本

**位置**：[HomePage.vue](packages/frontend/src/pages/HomePage.vue) 三个分区

**问题**：非遗、美食、产业三个分区的"了解更多"按钮链接到不同页面（`/culture`、`/food`、`/industry`），但屏幕阅读器看到的文本完全相同，无法区分。

**修复**：使用 `aria-label` 补充上下文：

```html
<a href="/culture" aria-label="了解更多非遗文化">了解更多</a>
<a href="/food" aria-label="了解更多万载美食">了解更多</a>
<a href="/industry" aria-label="了解更多烟花产业">了解更多</a>
```

## 三、P0 — 图片优化（影响最大、收益最高）

### 现状

- 66 张图片全部为 JPEG/PNG，无 WebP 格式
- 最大单张 `demo-home.png` 达 3.4MB
- 旅游照片平均 400KB-600KB
- 构建时无任何图片压缩流程
- 部分图片缺少 `loading="lazy"` 和 `decoding="async"`

### 方案

#### 3.1 构建时自动转 WebP + 压缩

安装 `vite-plugin-imagemin`：

```bash
pnpm --filter @wanzai/frontend add -D vite-plugin-imagemin
```

在 [vite.config.ts](packages/frontend/vite.config.ts) 中添加：

```ts
import viteImagemin from 'vite-plugin-imagemin'

// 在 plugins 数组中追加：
viteImagemin({
  gifsicle: { optimizationLevel: 7, interlaced: false },
  optipng: { optimizationLevel: 7 },
  mozjpeg: { quality: 80 },
  pngquant: { quality: [0.7, 0.85], speed: 4 },
  webp: { quality: 80 },
})
```

> **注意**：`vite-plugin-imagemin` 使用原生二进制，首次安装较慢。如果二进制安装失败，备选方案是构建前用 sharp 脚本批量处理。

#### 3.2 sharp 脚本（已实施）

> **关于画质与可逆性**：JPEG/PNG → WebP 是有损压缩，转换后丢失的信息无法恢复。但**原图始终保留在 `src/assets/images/`，不会被修改或删除**。WebP 文件输出到 `public/optimized/` 目录，页面通过 `<picture>` 标签优先加载 WebP，老浏览器回退到原图。如果对效果不满意，删除 `public/optimized/` 即可完全恢复。

采用保守参数：不缩放分辨率、PNG quality 85 / JPEG quality 80，视觉上与源文件基本无差异。若 WebP 体积 ≥ 原图则自动跳过（保留原格式）。

**脚本位置**：[packages/frontend/scripts/optimize-images.mjs](packages/frontend/scripts/optimize-images.mjs)

在 `scripts/optimize-images.mjs` 创建构建前脚本：

```js
// 批量：JPEG/PNG → WebP，保留原分辨率，原图不动
import sharp from 'sharp'
import { readdirSync, statSync, existsSync, mkdirSync } from 'fs'
import { resolve, extname, basename, dirname } from 'path'
import { fileURLToPath } from 'url'

const __dirname = dirname(fileURLToPath(import.meta.url))
const srcDir = resolve(__dirname, '../packages/frontend/src/assets/images')
const publicDir = resolve(__dirname, '../packages/frontend/public/optimized')

if (!existsSync(publicDir)) mkdirSync(publicDir, { recursive: true })

const files = readdirSync(srcDir).filter((f) => /\.(jpe?g|png)$/i.test(f) && !f.startsWith('.'))

for (const file of files) {
  const input = resolve(srcDir, file)
  const output = resolve(publicDir, basename(file, extname(file)) + '.webp')
  const originalSize = statSync(input).size

  await sharp(input).webp({ quality: 85 }).toFile(output)

  const optimizedSize = statSync(output).size
  const saved = ((1 - optimizedSize / originalSize) * 100).toFixed(0)
  console.log(
    `${file}: ${(originalSize / 1024).toFixed(1)}KB → ${(optimizedSize / 1024).toFixed(1)}KB (${saved}% saved)`,
  )
}
```

然后在 `package.json` 的 build 脚本中加入：

```json
"build": "tsc && node ../../scripts/generate-sitemap.js && node scripts/optimize-images.mjs && vite build && node ../../scripts/prerender.js"
```

#### 3.3 组件中使用 `<picture>` 标签

对关键大图使用 WebP 优先级加载 + JPEG fallback：

```html
<picture>
  <source srcset="/optimized/hero.webp" type="image/webp" />
  <img
    src="@/assets/images/hero.jpg"
    alt="万载烟花"
    loading="lazy"
    decoding="async"
    width="1200"
    height="800"
  />
</picture>
```

#### 3.4 首屏关键图加 `fetchpriority="high"`

首页 Hero 大图（LCP 元素）应加：

```html
<img src="..." fetchpriority="high" loading="eager" />
```

## 四、P1 — i18n 按需加载

### 现状

[locales/index.ts](packages/frontend/src/locales/index.ts) 包含全部中英文翻译，96KB 文本全部打入 main chunk（index.js 的 250KB 中占了 ~40%）。

### 方案

#### 4.1 拆分语言文件

创建三个文件：

**`locales/zh-CN.json`**（中文翻译）
**`locales/en.json`**（英文翻译）
**`locales/index.ts`**（仅保留 i18n 初始化逻辑）

```ts
// locales/index.ts
import { createI18n } from 'vue-i18n'

const i18n = createI18n({
  legacy: false,
  locale: 'zh-CN',
  fallbackLocale: 'zh-CN',
  messages: {
    'zh-CN': {}, // 空占位，运行时加载
    en: {},
  },
})

// 按需加载语言包
const loadedLocales = new Set<string>()

export async function loadLocale(locale: 'zh-CN' | 'en') {
  if (loadedLocales.has(locale)) return
  const messages =
    locale === 'zh-CN'
      ? (await import('./zh-CN.json')).default
      : (await import('./en.json')).default
  i18n.global.setLocaleMessage(locale, messages)
  loadedLocales.add(locale)
}

export default i18n
```

#### 4.2 路由守卫中触发加载

在 [router/index.ts](packages/frontend/src/router/index.ts) 的 `beforeEach` 中：

```ts
router.beforeEach(async (to, _from, next) => {
  const routeLocale = (to.meta.locale as string) || 'zh-CN'
  await loadLocale(routeLocale as 'zh-CN' | 'en')
  i18n.global.locale.value = routeLocale as 'zh-CN' | 'en'
  localStorage.setItem('locale', routeLocale)
  document.title = (to.meta.title as string) || '焰境·万载'
  next()
})
```

### 预期收益

| 指标                    | 优化前 | 优化后               |
| ----------------------- | ------ | -------------------- |
| main chunk 中 i18n 占比 | ~40KB  | ~2KB（仅初始化逻辑） |
| 首屏 JS 总量            | 250KB  | ~210KB               |

## 六、P2 — 字体加载优化

### 现状

- Google Fonts 通过 preconnect 加载
- URL 中未指定 `display=swap`，文字在字体加载期间不可见（FOIT）

### 方案

在 [index.html](packages/frontend/index.html) 和构建后 HTML 中，加载 Google Fonts 时显式指定：

```html
<!-- 当前 -->
<link rel="preconnect" href="https://fonts.googleapis.com" />

<!-- 优化后：在 CSS 中加载时加 display=swap -->
<!-- 在 style.css 或对应的 <style> 中 -->
@import url('https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@400;500;700&display=swap');
```

或者在 JS 中通过 `document.fonts.ready` 确保字体加载后再渲染关键文字。

## 七、P2 — 资源提示（Resource Hints）

### 现状

- 有 preconnect 到 Google Fonts 和 AMap
- 缺少对关键 JS/CSS chunk 的 `preload`
- 缺少对关键字体文件的 `preload`

### 方案

在 [scripts/prerender.js](scripts/prerender.js) 中为每个预渲染 HTML 添加：

```html
<!-- 预加载 Vue 生态链（首屏必需） -->
<link rel="modulepreload" href="/assets/vendor-vue-xxxxx.js" />
<link rel="preload" href="/assets/index-xxxxx.css" as="style" />

<!-- 预加载字体（如果使用自定义字体） -->
<link
  rel="preload"
  href="/fonts/NotoSansSC-Regular.woff2"
  as="font"
  type="font/woff2"
  crossorigin
/>
```

注意 chunk hash 在每次构建后会变化，需要在构建脚本中动态解析 `dist/assets/` 下的文件名。

## 八、P2 — 安全头完善

### 8.1 开发环境补全 Permissions-Policy

在 [vite.config.ts](packages/frontend/vite.config.ts) 的 `server.headers` 中添加：

```ts
'Permissions-Policy': 'camera=(), microphone=(), geolocation=(self)',
```

目前仅在 `preview.headers` 中有此配置。

### 8.2 审计发现的安全问题

| 问题                            | 影响页面 | 说明                                                            |
| ------------------------------- | -------- | --------------------------------------------------------------- |
| **HSTS 头缺失**                 | 所有页面 | 无 `Strict-Transport-Security`，存在降级攻击风险                |
| **COOP 头缺失**                 | 所有页面 | 无 `Cross-Origin-Opener-Policy`，popup 可访问 window.opener     |
| **Trusted Types 未配置**        | 所有页面 | DOM XSS 防护不足                                                |
| **CSP `unsafe-inline`**         | 所有页面 | `script-src` 包含 `'unsafe-inline'`（Tailwind 需要）            |
| **CSP `strict-dynamic` 未使用** | 所有页面 | 建议配合 nonce/hash 使用 `'strict-dynamic'` 替代 host allowlist |

### 8.3 生产 Nginx 配置建议

```nginx
# 在 server 块中添加
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header Permissions-Policy "camera=(), microphone=(), geolocation=(self)" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-Frame-Options "SAMEORIGIN" always;  # 或 DENY
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Cross-Origin-Opener-Policy "same-origin" always;

# Gzip 压缩（如果 Nginx 层未开）
gzip on;
gzip_vary on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
gzip_min_length 1024;
gzip_comp_level 6;

# Brotli 压缩（如果编译了 ngx_brotli 模块）
brotli on;
brotli_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
brotli_comp_level 6;

# 静态资源缓存
location /assets/ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}

location ~* \.(jpg|jpeg|png|webp|svg|ico|woff2)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

> **注意**：CSP `unsafe-inline` 是 Tailwind CSS 的已知依赖。生产环境可考虑用 nonce 方式生成 CSP，将 Tailwind 的 inline style 安全化。短期可接受现状，长期列入 P3。

## 九、P3 — 低优先级优化

### 9.1 移除 `X-UA-Compatible` meta

[packages/frontend/index.html](packages/frontend/index.html) 中的这行已过时：

```html
<!-- 可删除 -->
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
```

IE11 已于 2022 年退役，现代浏览器忽略此 meta。

### 9.2 `chart.js` 按需加载

`chart.js` 只在 [IndustryPage.vue](packages/frontend/src/pages/IndustryPage.vue) 中使用（1 个页面），且该页面已经是动态导入。当前 manualChunks 将其单独分割为 `vendor-chart.js`（207KB），这是正确的 — 它不会在首屏加载。

但如果 IndustryPage 的某些子组件被其他页面引用，需要确认 chart.js 不会泄漏到主包。验证方式：

```bash
grep -r "chart.js\|Chart" packages/frontend/src --include="*.vue" --include="*.ts" -l
# 应只有 IndustryPage.vue
```

### 9.3 `three.js` 确认未打包

`ThreeAiModel.vue` 中 import 了 three.js（~500KB），但该组件当前被注释掉（`AIChatButton.vue` 中 `// defineAsyncComponent`）不会进入构建。如果未来启用，务必保持 `defineAsyncComponent` 方式加载。

### 9.4 图片 `width` / `height` 属性

为所有 `<img>` 标签添加 `width` 和 `height` 属性，减少 CLS（布局偏移）：

```html
<img src="..." width="800" height="600" loading="lazy" />
```

浏览器会根据宽高比预留空间，图片加载完成后不会撑开布局。**CLS 从 Lighthouse 模拟检测为 0.674（差），此修复是降低 CLS 的关键手段。**

### 9.5 首页大型背景视频

如果首页有 Hero 视频背景：

- 使用 `<video autoplay muted loop playsinline>` + `poster` 属性
- 视频文件压缩到 < 2MB
- 移动端用 poster 静态图替换视频
