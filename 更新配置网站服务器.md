# 网站服务器 — 旧架构迁移至新架构

## 当前状态 vs 目标状态

| 组件 | 当前（旧） | 目标（新） |
|------|-----------|-----------|
| 后端框架 | Express (`server/index.js`) | NestJS (`packages/backend/`) |
| 数据库 | 无（硬 coded 知识库） | PostgreSQL 16 + Prisma |
| 后端端口 | 3001（但 Nginx 代理到 8080） | 8080（沿用 Nginx 配置） |
| 后端目录 | `/usr/share/nginx/server/` | `/usr/share/nginx/wanzai-backend/` |
| pm2 进程 | `whizz-backend` | `wanzai-backend` |
| 前端目录 | `/usr/share/nginx/html/dist/` | 不变 |
| SSL 证书 | `/usr/share/nginx/ssl/` | 不变 |

## 一键迁移步骤

以下命令全部在服务器上以 `root` 执行。

### 0. 前置准备

```bash
# 安装 rsync（CI 部署需要）
dnf install -y rsync --disablerepo=mongodb-org-6.0

# 可选：修复损坏的 MongoDB 源（避免后续 dnf 报错）
rm -f /etc/yum.repos.d/mongodb-org-6.0.repo
```

### 1. 安装 PostgreSQL 16

```bash
# 添加 PostgreSQL 官方源
dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-8-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# 禁用系统自带模块
dnf -qy module disable postgresql

# 安装
dnf install -y postgresql16-server

# 初始化并启动
/usr/pgsql-16/bin/postgresql-16-setup initdb
systemctl enable postgresql-16
systemctl start postgresql-16
```

### 2. 创建数据库和用户

```bash
# 创建应用用户（请替换 <your-password>）
sudo -u postgres psql <<'SQL'
CREATE USER wanzai WITH PASSWORD '<your-password>';
CREATE DATABASE wanzai OWNER wanzai;
GRANT ALL PRIVILEGES ON DATABASE wanzai TO wanzai;
\c wanzai
GRANT ALL ON SCHEMA public TO wanzai;
SQL
```

### 3. 创建后端工作目录和 .env

```bash
mkdir -p /usr/share/nginx/wanzai-backend

cat > /usr/share/nginx/wanzai-backend/.env <<'EOF'
# 数据库
DATABASE_URL=postgresql://wanzai:<your-password>@localhost:5432/wanzai

# JWT
JWT_SECRET=<random-64-char-string>

# Redis
REDIS_URL=redis://localhost:6379

# LLM API（注意变量名必须用 API_KEY 和 BASE_URL，代码里是固定读这两个名）
API_KEY=sk-rRYx47NFc9GqoiHVKv-Wow
BASE_URL=https://llmapi.paratera.com
MODEL_NAME=MiniMax-M2.5

# 端口（匹配 Nginx proxy_pass）
PORT=8080

# CORS 允许的源
ALLOWED_ORIGINS=https://whizzzest-yanjingwanzai.top,http://localhost:5173
EOF
```

> `.env` 包含敏感信息，不会被 CI 部署覆盖。变量说明详见下方的「环境变量参考」。

### 4. 完成首次部署

**方式一：手动（推荐首次）**

```bash
# 本地构建后端
cd /path/to/whizzzest\(website-version\)
pnpm build:backend

# 准备部署包
cd packages/backend
mkdir -p deploy
cp -r dist deploy/
cp -r prisma deploy/
sed 's/"@wanzai\/contracts": "workspace:\*",//' package.json > deploy/package.json

# 上传到服务器
scp -i ~/Downloads/焰境万载.pem -r deploy/* root@whizzzest-yanjingwanzai.top:/usr/share/nginx/wanzai-backend/

# SSH 到服务器安装依赖并启动
ssh -i ~/Downloads/焰境万载.pem root@whizzzest-yanjingwanzai.top <<'REMOTE'
cd /usr/share/nginx/wanzai-backend
npm install --prod
npx prisma generate
npx prisma migrate deploy
npx prisma db seed
pm2 start dist/main.js --name wanzai-backend
pm2 save
REMOTE
```

**方式二：通过 GitHub Actions**

push 到 `main` 分支后自动触发。首次需要确保：
- GitHub Secrets 中已配置 `SSH_PRIVATE_KEY`、`SSH_HOST`、`SSH_USER`
- 服务器上已创建好 `/usr/share/nginx/wanzai-backend/.env`

### 5. 停止旧服务

```bash
# 确认新服务正常后，停止旧 Express 服务
pm2 stop whizz-backend
pm2 delete whizz-backend
pm2 save

# 可选：删除旧代码
rm -rf /usr/share/nginx/server
```

### 6. 验证

```bash
# pm2 状态
pm2 status

# 后端健康
curl http://localhost:8080/api/v1/

# 前端
curl -s -o /dev/null -w "%{http_code}" https://whizzzest-yanjingwanzai.top

# 数据库
sudo -u postgres psql -d wanzai -c "\dt"
```

## 日常部署（自动）

push `main` 后 `.github/workflows/deploy.yml` 自动执行：

1. 构建前端 + 后端（Node 24，pnpm workspace）
2. rsync 前端 dist → `/usr/share/nginx/html/dist/`（**排除** `assets/videos` 保护视频）
3. rsync 后端 dist + prisma → `/usr/share/nginx/wanzai-backend/`（分目录同步，保护 `.env`）
4. `npm install --prod` → `prisma generate` → `prisma migrate deploy`
5. `pm2 reload`（已存在）或 `pm2 start`（首次）
6. 健康检查

### GitHub Secrets 清单

| Name | 说明 |
|------|------|
| `SSH_PRIVATE_KEY` | `焰境万载.pem` 文件的**完整内容** |
| `SSH_HOST` | `whizzzest-yanjingwanzai.top` |
| `SSH_USER` | `root` |
| `VITE_AMAP_KEY` | 高德地图 Key |
| `VITE_AMAP_SECURITY_CODE` | 高德地图安全密钥 |

## 环境变量参考

服务器上的 `/usr/share/nginx/wanzai-backend/.env`（不会被 CI 覆盖）：

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 `postgresql://user:pass@localhost:5432/wanzai` |
| `JWT_SECRET` | JWT 签名密钥，建议 `openssl rand -hex 32` 生成 |
| `REDIS_URL` | Redis 连接串，默认 `redis://localhost:6379` |
| `API_KEY` | LLM API 密钥 |
| `BASE_URL` | LLM 代理地址（不含 `/v1`） |
| `MODEL_NAME` | 模型名称 `MiniMax-M2.5` |
| `PORT` | 后端端口 `8080`（匹配 Nginx `proxy_pass`） |
| `ALLOWED_ORIGINS` | CORS 允许的域名，逗号分隔 |

## 视频文件管理

视频文件（约 510 MB）未纳入 Git。本地开发和线上部署通过不同方式处理：

| 场景 | 路径 | 说明 |
|------|------|------|
| 本地 dev | `src/assets/videos/` | symlink `public/assets/videos` 指向此处 |
| 线上 prod | `/usr/share/nginx/html/dist/assets/videos/` | 手动上传，CI 保护不删 |

### 首次上传到服务器

```bash
scp -i ~/Downloads/焰境万载.pem \
  packages/frontend/src/assets/videos/*.mp4 \
  root@whizzzest-yanjingwanzai.top:/usr/share/nginx/html/dist/assets/videos/
```

上传后验证：

```bash
ssh -i ~/Downloads/焰境万载.pem root@whizzzest-yanjingwanzai.top \
  "ls -lh /usr/share/nginx/html/dist/assets/videos/"
```

### 后续更新

每次 CI 部署时 `rsync --exclude=assets/videos` 跳过视频目录，服务器上的视频不会被删除。新增或替换视频手动 `scp` 即可，无需重新部署。

## SSL 证书更新

证书文件路径（不变）：

| 文件 | 路径 |
|------|------|
| 证书 | `/usr/share/nginx/ssl/whizzzest-yanjingwanzai.top.pem` |
| 密钥 | `/usr/share/nginx/ssl/whizzzest-yanjingwanzai.top.key` |

```bash
# 替换证书后重载 Nginx
sudo cp new-cert.pem /usr/share/nginx/ssl/whizzzest-yanjingwanzai.top.pem
sudo cp new-key.key /usr/share/nginx/ssl/whizzzest-yanjingwanzai.top.key
sudo nginx -t && sudo systemctl reload nginx

# 验证
openssl s_client -connect whizzzest-yanjingwanzai.top:443 -servername whizzzest-yanjingwanzai.top </dev/null 2>/dev/null | openssl x509 -noout -dates
```
